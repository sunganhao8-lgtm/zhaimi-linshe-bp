## 🔥 关键转折点

### 1. 从"几何直线"到"真实步行路径"

**一开始**：地图上从店到万达画一条直线（数学上正确，但现实不可走）。

**问题**：直线穿楼、过墙、不可达。

**解决**：用高德 `AMap.Walking` API 实测真实步行路径（29 点 polyline / 932m / 12 分钟），固化到 `data/poi.json` 永久本地。

**教训**：**几何距离 ≠ 实际可达距离**。选址分析必须用真实步行/驾车 API。

### 2. 从"65㎡"到"49㎡"

**一开始**：BP 写 65㎡（业主口头报的"实用面积"）。

**实测**：卷尺量下来只有 49㎡（7m×7m 实际可用空间）。

**修正**：BP / 布局编辑器 / README 三处全部同步改成 49㎡。

**教训**：**所有数字必须实测**，二手数据不写入交付物。

### 3. 从"云 API 依赖"到"全本地"

**一开始**：每次打开 `site-map.html` 都要调高德 API。

**问题**：
- 演示时网络不稳定
- API 配额限制
- 凭据泄漏风险

**解决**：把 11 POI 坐标 + 步行路径 polyline 固化到 `data/poi.json`（4427 字节），离线也能用。

**教训**：**交付物应该自包含**，不依赖外部服务才能稳定演示。

### 4. 从"组件随便拖"到"防碰撞 + 边界保护"

**一开始**：用户可以把组件拖到画布外面 / 重叠。

**问题**：实际开店不会接受错误布局。

**解决**：
- 边界 clamp：拖到画布边自动停住
- 碰撞检测：AABB 矩形比较，重叠时推到不重叠位置
- 墙组件特殊：不参与碰撞阻挡（普通组件撞墙会反弹）

**教训**：**编辑器要有约束**，否则用户做出来的方案不能落地。
