## DEC-001: 选址在临港泥城（万达东侧 500m）

**日期**：2026-06-16

**背景**：
- 创始人在临港泥城片区考察多个转让铺面
- 需要同时满足：成熟商圈（步行可达）/ 住宅密集（养宠人群）/ 转让费可控（<10w）

**选项**：
- ❌ 浦东金桥 / 静安寺：转让费 30w+、竞争激烈、客群偏年轻但养宠率低
- ❌ 临港主城区（申港大道）：人流量大但租金 15w/年，回本周期拉长
- ❌ 临港泥城（万达周边）：住宅密集（6 个小区 1km 内）、租金 8w/年、转让费 9-10w
- ✅ **临港泥城·万达东侧**：成本可控 + 客群匹配 + 步行可达

**选定**：临港泥城片区，万达广场东侧 500m 的 49㎡ 商铺。

**影响**：
- 启动资金压到 22 万
- 步行 1km 覆盖 6 个小区，潜在客户 3000+ 户
- 3km 辐射人口 30 万（含临港主城区）

**修订**：无

## DEC-002: 接手 49㎡（7m×7m）门店

**日期**：2026-06-16

**背景**：
- 业主口头报"实用面积 65㎡"
- 实际现场勘察：可用空间只有 49㎡（7m×7m 主厅 + 1m 走道 + 卫生间）
- 需要在 BP 和布局编辑器之间选一个数字

**选项**：
- ❌ 写 65㎡：BP 看起来"大店"，但实际做不到，交付物与现实脱节
- ✅ **写 49㎡**：BP / 布局编辑器 / README 三处保持一致

**选定**：49㎡（7m×7m），所有交付物同步。

**影响**：
- 30 个组件必须塞进 7×7 空间
- 干湿分区设计：前 5m 干区（沙发/货架/收银），后 5m 湿区（洗护池/烘干/美容）
- 单店坪效目标提升（49㎡ 必须做出 60㎡ 的产能）

**教训**：**所有数字实测**，二手数据不写进交付物。

**修订**：无

## DEC-003: 转让费 9-10w（含 2w 押金）

**日期**：2026-06-16

**背景**：
- 业主报价：转让费 9 万 + 押金 2 万 = 11 万
- 装修预算原本 6 万，被转让费挤掉一半

**选项**：
- ❌ 砍转让费：业主不会让步，已经比同片区低
- ❌ 加预算：超出 22w 启动资金上限
- ✅ **接受 9-10w（含 2w 押金）**：装修降到 4w（简装）

**选定**：转让费 9-10w（含 2w 押金），装修预算调到 4-6w。

**影响**：
- 启动资金重新分配：转让 9-10w / 装修 4-6w / 设备 4w / 流动 2w = 总 19-22w
- 装修方案简化：自己刷墙 + 二手设备 + 简易吊顶
- 风险：押金 2w 退租时可能扯皮，需要合同写清楚

**修订**：无

## DEC-004: 月固定成本 1.57 万 / 回本 24-32 月

**日期**：2026-06-16

**背景**：
- 房租：8w/年 = 6700/月
- 人力：1 全职（店长兼洗护师）+ 1 兼职 = 6500/月
- 水电：1000/月
- 杂费（耗材 / 物业 / 推广）：1500/月
- 合计 1.57 万/月固定成本

**回本计算**：
- 平均客单价 80-120 元
- 日均 8-10 单 = 月 240-300 单
- 月营收 1.9-3.6 万
- 净利 0.4-2.0 万/月
- 回本 22w / 平均净利 1.2w ≈ 18 个月（乐观）
- 保守估计 24-32 个月

**选项**：
- ❌ 写 12-18 月回本：太乐观，投资人不信
- ❌ 写 36+ 月回本：太悲观，影响决策
- ✅ **写 24-32 月**：符合宠物店行业平均水平

**选定**：24-32 个月回本。

**影响**：
- 投资人看到合理预期，不会过度乐观
- 现金流压力测算：第 6 个月开始正向现金流，第 24 个月回本

**修订**：无

## DEC-005: 数据全部本地化（不依赖云 API）

**日期**：2026-06-17

**背景**：
- 一开始设计：每次打开 `site-map.html` 都调高德 API 实时获取 POI
- 演示时遇到：网络不稳定 / API 配额限制 / 凭据泄漏风险

**选项**：
- ❌ 实时调用 API：体验差、有配额限制、有凭据风险
- ✅ **离线本地缓存**：把 11 POI 坐标 + 步行路径 polyline 固化到 `data/poi.json`

**选定**：所有数据本地缓存，离线可演示。

**影响**：
- `data/poi.json` 4427 字节，包含 11 POI + 932m 步行路径
- 高德 API 仅用于首次数据采集，之后不再调用
- 演示环境无网络也能跑

**教训**：**交付物应该自包含**，不依赖外部服务。

**修订**：无

## DEC-006: 布局编辑器加碰撞检测 + 边界保护

**日期**：2026-06-17

**背景**：
- 一开始：用户可随意拖组件，能拖到画布外、能重叠
- 问题：实际开店不接受错误布局

**选项**：
- ❌ 不加约束：用户自由度高，但做出来的方案不能落地
- ❌ 软提示（红框警告）：用户还是会犯错
- ✅ **硬约束**：边界 clamp + 碰撞自动推开

**选定**：拖动时实时检测，碰撞自动推到不重叠位置。

**影响**：
- 实现：`rectsOverlap()` AABB 矩形比较 + `resolveCollision()` 4 方向推动
- 墙组件特殊：不参与碰撞阻挡（普通组件撞墙会反弹）
- 移动端 rAF 节流，避免拖动卡顿

**教训**：**编辑器要有约束**，否则用户做出来的方案不可用。

**修订**：无

## DEC-007: 高德地图 + 本地 OSM 瓦片混合方案

**日期**：2026-06-17

**背景**：
- 纯高德地图：每次都要联网，加载慢
- 纯 OSM 瓦片：免费但需要自己下载缓存

**选项**：
- ❌ 纯高德：体验好但依赖网络
- ❌ 纯 OSM：免费但首次加载慢
- ✅ **混合**：高德 JS API（标记 / 路径 / InfoWindow）+ 本地 OSM 瓦片（底图）

**选定**：高德 JS API 负责交互层，本地 OSM 瓦片负责底图渲染。

**影响**：
- 首次打开快（本地瓦片直接渲染）
- 交互功能完整（高德负责）
- **坑**：本地瓦片最高 z=16，zoom>17 会出现白板 → `zooms: [11, 17]` 强制限制

**教训**：**混合方案要明确各自职责**，避免冲突。

**修订**：无

## DEC-008: 公开 GitHub 仓库（MIT License）

**日期**：2026-06-18

**背景**：
- 创始人想：项目交付物是否要公开？
- 风险：BP 含商业机密（成本 / 回本周期 / 客群分析）

**选项**：
- ❌ 完全私有：安全，但失去外部贡献 / 信用背书
- ❌ 部分公开：管理复杂
- ✅ **完全公开（MIT）**：商业机密在 BP.md 详细版，repo 只放结构和数据

**选定**：public repo + MIT License。

**影响**：
- URL：`https://github.com/sunganhao8-lgtm/zhaimi-linshe-bp`
- 在线预览：`https://sunganhao8-lgtm.github.io/zhaimi-linshe-bp/`
- 详细 BP 在 `bp.md`（合伙人私下传，不在投资人 deck 里）
- 对外 deck 只放 `bp.html`（PPT 形式）

**风险缓解**：
- Repo 公开的是**方法论 + 工具**（选址地图 / 布局编辑器）
- 不公开的是**具体财务数字**（在 bp.md，私下传）
- 不公开的是**业主信息**（坐标已脱敏到城市级）

**修订**：无

## DEC-009: 合伙人用 Collaborator 角色（不开组织）

**日期**：2026-06-18

**背景**：
- 合伙人要加入维护
- 选项：直接加 Collaborator vs 开 GitHub Organization

**选项**：
- ❌ 开 Organization：管理成本高（要管 team / 权限 / 计费），项目太小不需要
- ✅ **直接加 Collaborator**：单 repo 协作够用

**选定**：Settings → Collaborators → Add people。

**影响**：
- 合伙人能直接 push main / 创建分支 / 提 PR / 关 Issue
- 不需要 fork 流程
- 适合 2-3 人小团队

**修订**：如果团队超过 5 人，迁移到 Organization。

## DEC-010: 加电竞展示区（展示型非经营型）

**日期**：2026-06-18

**决策**：在门店右手边（沿玻璃面）14㎡ 设置"电竞展示区"，配置 1 台 PS5 + 1 台 Switch + 2 显示器 + 3 电竞椅 + 吧台+氛围灯，**总计 1.47 万投入，0 元/月运营成本**。

**为什么加**：

1. **差异化引流**——临港泥城 6 家竞品全部是纯宠物店，无任何"宠物+电竞"复合场景
2. **等位体验升级**——洗护 30-60 分钟等位时间，顾客可免费玩主机游戏，提升满意度
3. **小红书内容素材**——"宠友+电竞"主题稀缺，差异化传播
4. **不增加月固定成本**——不雇人 / 不收费 / 不通宵 / 复用既有水电

**为什么不经营**：

1. 主业是宠物洗护，**电竞是装饰品不是第二收入曲线**
2. 经营需要额外人手、电费、夜间安保、月营收难以覆盖成本
3. 店主精力有限，专注洗护核心业务

**财务影响**：

- 启动资金从 22 万 → **23.5 万**（+1.5 万电竞设备）
- 月固定成本不变（1.57 万/月）
- 回本周期不变（24-32 个月）
- 月固定资金从 3.0 万 → 1.8 万（电竞 1.5 万从中出）

**空间安排**（基于方案 A 默认假设：转角铺·南门+西墙玻璃，待现场核对）：

- 右手边（北-东）14㎡ 作为电竞体验区
- 沿西墙玻璃面放寄养笼（自然光，宠物舒适）
- 入门左手边柜子（约 1.5m 宽）作为视觉屏障

**风险**：

1. 玻璃面/门朝向假设错误 → layout-editor 加 5 个电竞组件，让用户在编辑器内自由调整
2. 电竞元素喧宾夺主 → 视觉区分（紫色 isEsport 标签 vs 暖橙 isPet 标签）
3. 顾客只看不消费 → 拍照打卡也是消费（小红书传播价值 ≥ 等额现金）

## DEC-011: 启动资金 22→23.5 万（电竞设备从流动资金出）

**日期**：2026-06-18

**决策**：将流动资金从 3.0 万 → 1.8 万（电竞 1.5 万从中出），启动资金从 22 万 → 23.5 万。

**为什么这样分**：

- 维持 9 项明细表的清晰性，新增"电竞设备"作为独立第 5 项
- 流动资金从 3.0 → 1.8 仍保留 0.8 万应急储备金
- 明细合计 26.5 万（理想配置），实际启动预算上限 23.5 万（砍营销/囤货/流动超支）

**修订**：明细合计与实际启动预算不严格相等是 BP 行业常态，详细分项是"理想配置"，实际启动是"必需投入"。合伙人看到 22 万 → 23.5 万时不要惊讶，是新增电竞 1.5 万。
